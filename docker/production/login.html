Authentication Instructions

  • Access is granted via a magic link in the format: /login?token=SECRET_KEY
  • The SECRET_KEY is configured by you when you have started the docker container
  • If you have access to an already logged-in device, you can generate a QR code or copy the login link from within the app
  • Once authenticated, a secure session cookie will be set for continued access
  • All application routes and API endpoints are protected by this simple authentication
